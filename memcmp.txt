1. Базовые сценарии

1.1. Равные строки

- Описание: Сравнение двух одинаковых строк.
- Входные данные:
    - str1 = "abc"
    - str2 = "abc"
    - n = 3
- Ожидаемый результат: 0 (строки равны).

1.2. Разные строки (первая меньше)

- Описание: Сравнение строк, где str1 меньше str2 (по значению первого отличающегося байта).
- Входные данные:
    - str1 = "abc"
    - str2 = "abd"
    - n = 3
- Ожидаемый результат: Отрицательное число (первая строка меньше).

1.3. Разные строки (первая больше)

- Описание: Сравнение строк, где str1 больше str2 (по значению первого отличающегося байта).
- Входные данные:
    - str1 = "abd"
    - str2 = "abc"
    - n = 3
- Ожидаемый результат: Положительное число (первая строка больше).

2. Граничные случаи

2.1. Нулевой размер (n = 0)

- Описание: Сравнение строк с нулевым размером.
- Входные данные:
    - str1 = "abc"
    - str2 = "def"
    - n = 0
- Ожидаемый результат: 0 (ничего не сравнивается).

2.2. Разный размер строк

- Описание: Сравнение строк, где одна строка короче другой, но n меньше или равен длине более короткой строки.
- Входные данные:
    - str1 = "abc"
    - str2 = "ab"
    - n = 2
- Ожидаемый результат: 0 (первые 2 байта равны).

2.3. Сравнение с нулевым байтом

- Описание: Сравнение строк, содержащих нулевой байт (\0).
- Входные данные:
    - str1 = "ab\0c"
    - str2 = "ab\0d"
    - n = 4
- Ожидаемый результат: 0 (первые 3 байта равны, включая нулевой байт).

3. Специальные символы и бинарные данные

3.1. Сравнение бинарных данных

- Описание: Сравнение бинарных данных (не текстовых строк).
- Входные данные:
    - str1 = "\x01\x02\x03"
    - str2 = "\x01\x02\x04"
    - n = 3
- Ожидаемый результат: Отрицательное число (первый отличающийся байт в str1 меньше).

3.2. Сравнение с символами вне ASCII

- Описание: Сравнение строк, содержащих символы вне диапазона ASCII.
- Входные данные:
    - str1 = "café"
    - str2 = "café"
    - n = 5
- Ожидаемый результат: 0 (строки равны).

4. Ошибочные случаи

4.1. Передача NULL в качестве указателя

- Описание: Сравнение с NULL-указателем.
- Входные данные:
    - str1 = NULL
    - str2 = "abc"
    - n = 3
- Ожидаемый результат: Ошибка (поведение зависит от реализации, но обычно это приводит к краху).

4.2. Передача невалидного размера

- Описание: Сравнение с размером, превышающим длину строк.
- Входные данные:
    - str1 = "abc"
    - str2 = "abc"
    - n = 10
- Ожидаемый результат: 0 (если память за пределами строк инициализирована нулями), или неопределенное поведение.

5. Дополнительные сценарии

5.1. Сравнение длинных строк

- Описание: Сравнение длинных строк (например, 1 МБ данных).
- Входные данные:
    - str1 = "очень_длинная_строка_1"
    - str2 = "очень_длинная_строка_2"
    - n = длина строки
- Ожидаемый результат: Зависит от данных (может быть 0, отрицательное или положительное число).

5.2. Сравнение строк с повторяющимися символами

- Описание: Сравнение строк, где символы повторяются.
- Входные данные:
    - str1 = "aaaaa"
    - str2 = "aaaab"
    - n = 5
- Ожидаемый результат: Отрицательное число (первая строка меньше).
